Release Plan
UCSC Bus Buddy
Team SPARC
Deadline: December 9th, 2013
Version 1.2
Revision Date: 10/08/2013

To create an android app designed for UCSC students using school and
Santa Cruz Metro buses.

High level goals:
1. Have bus stop icons on a map overly
   Have the Santa Cruz Metro bus schedule
2. Have a main page for the app
   Create a user manual for the app
3. Have an option for the next closest bus stop at current location
   Completely test the app’s functions and usability
   Design graphics for the icons for the app

User stories:
Sprint I:
As a student user, I want to know the closest bus stop to me, so that I
can quickly catch the closest bus to me. - 5 points

As a student user, I want to have the entire bus schedule, so that I
coordinate my schedule around the bus schedule. - 1 points

As a developer, I want to build a basic interface so that I can develop
basic functionality. - 8 points

Sprint II:
As a student user, I want an interface to navigate on the app, so that
I can find the option that best fits my need. - 5 points

As a student user, I must be able to read a user manual of the app. - 3 points

Sprint III:
As a student user, I want to know when the next bus is coming at my
current location, so I can determine whether I want to wait for the bus now. - 8 points

As a tester, I need to be able to test all of the functions implemented
and make sure they are working properly. - 5 points

As a student user, I want each option to have an icon, so that I can
associate a graphic with a function of the app. - 3 points

Product Backlog:
- Implement mapping of loop buses currently on UCSC campus.

As a student user, I want to know where the loop buses are at, so that
I would know whether or not I should walk to my destination. - 13 points
